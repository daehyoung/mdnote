# 작업 수행 및 문서 동기화 워크플로우

이 워크플로우는 새로운 작업을 시작할 때 계획을 수립하고, 작업 완료 후 관련 문서를 실시간으로 업데이트하여 코드와 문서의 동기화를 유지하는 것을 목적으로 합니다.

## 1. 작업 계획 수립 (Planning Phase)
- **목표**: 구현 전 핵심 설계 결정 및 영향도 분석.
- **활동**: 
    - `docs/worklogs/` 디렉토리에 날짜와 작업명을 포함한 구현 계획서(`YYYYMMDD_task_name.md`)를 작성합니다.
    - `current_tasks.md`의 `[Task List]`와 `[Active Files]`를 업데이트하여 현재 진행 상태를 명시합니다.
- **도구**: `write_to_file`, `replace_file_content`

## 2. 기능 구현 (Execution Phase)
- **목표**: 계획에 따른 코드 작성 및 로컬 검증.
- **활동**:
    - 구현 계획서에 명시된 `[Proposed Changes]`를 바탕으로 코드를 수정합니다.
    - 수정 중 설계 변경이 필요한 경우 구현 계획서와 `task.md`를 즉시 업데이트합니다.
- **도구**: `multi_replace_file_content`, `run_command`

## 3. 문서 동기화 및 아카이빙 (Documentation Phase)
- **목표**: 변경 사항을 공식 문서에 반영하고 기록 보존.
- **활동**:
    - **README.md 업데이트**: 새로운 주요 기능이나 문서가 추가된 경우 `README.md`의 관련 섹션에 링크를 추가합니다.
    - **기술서 업데이트**: `docs/` 내의 관련 상세 문서(예: `02_feature_analysis.md`, `04_frontend_ui_manual.md` 등)를 최신 상태로 갱신합니다.
    - **결과 문서 아카이빙**: 작업 완료 후 생성된 `walkthrough.md`를 `docs/worklogs/YYYYMMDD_task_name_walkthrough.md`로 복사하여 저장합니다.
    - **README.md 업데이트**: `README.md`의 작업 로그 섹션에 해당 계획과 결과를 모두 링크합니다.
- **도구**: `replace_file_content`, `multi_replace_file_content`

## 4. 최종 검증 및 종료 (Verification Phase)
- **목표**: 전체 시스템 안정성 확인 및 상태 요약.
- **활동**:
    - `current_tasks.md`를 업데이트하여 모든 작업을 완료 상태( [x] )로 표시합니다.
    - 배포 스크립트(`deploy.sh`)와 통합 테스트를 실행하여 정상 작동 여부를 확인합니다.
    - 사용자에게 `walkthrough.md`를 통해 최종 수행 결과를 보고합니다.
    - **Git Commit & Push**: 모든 변경 사항(코드 및 문서)을 커밋하고 원격 저장소에 푸시합니다.
        ```bash
        git add .
        git commit -m "feat: [작업명] 구현 및 관련 문서 업데이트"
        git push origin [branch-name]
        ```
- **도구**: `current_tasks.md` 업데이트, `walkthrough.md`, `git`
