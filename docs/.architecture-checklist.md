
## 📋 소프트웨어 아키텍처 설계 의사결정 체크리스트

### 1. 비즈니스 맥락 및 규모 (Scalability & Performance)

* [ ] **대상 사용자 및 트래픽:** 주요 타겟층은 누구이며, 예상되는 **평균/피크 TPS**는 얼마인가?
* [ ] **데이터 성장성:** 초기 데이터량과 월간/연간 예상 **데이터 증가분**은 어느 정도인가?
* [ ] **지리적 범위:** 서비스 지역이 단일 국가인가, 아니면 **글로벌 멀티 리전** 대응이 필요한가?

### 2. 가용성 및 신뢰성 (Availability & Reliability)

* [ ] **복구 목표 (RTO;Recovery Time Objective/RPO; Recovery Point Objective):** 허용 가능한 최대 서비스 중단 시간과 데이터 복구 지점은 어디인가?
* [ ] **SPOF(Single Point of Failure) 제거:** 시스템 내에 **단일 장애 지점**이 존재하는가? (DB, 로드밸런서, 특정 API 등)
* [ ] **장애 전파 방지:** 외부 시스템 장애 시 전체 시스템 붕괴를 막을 **서킷 브레이커**가 설계되었는가?
* [ ] **우아한 성능 저하:** 부분 장애 발생 시 핵심 기능 유지를 위한 **기능 격리** 전략이 있는가?

### 3. 상태 관리 및 서버 확장성 (State & Scale)

* [ ] **무상태성 (Stateless):** 애플리케이션 서버가 **Stateless**인가? 로컬 메모리에 상태를 저장하는가?
* [ ] **세션 관리:** 세션 정보를 **공유 저장소(Redis 등)**에 저장하여 수평 확장(Scale-out)이 가능한가?
* [ ] **Sticky Session:** 특정 서버에 연결을 고정해야 하는 **Stateful**한 비즈니스 로직이 존재하는가?

### 4. 배포 및 인프라 전략 (Deployment & DevOps)

* [ ] **배포 환경:** 클라우드(AWS/Azure 등), 온프레미스, 혹은 하이브리드 중 어디에 배포하는가?
* [ ] **무중단 배포:** 서비스 중단 없는 업데이트를 위해 **Blue-Green**이나 **Canary** 배포를 지원하는가?
* [ ] **롤백 자동화:** 배포 실패 시 즉시 이전 버전으로 복구할 수 있는 **자동화된 프로세스**가 있는가?
* [ ] **IaC 도입:** 인프라 구성을 코드로 관리(Terraform/CloudFormation)하여 환경 일치성을 보장하는가?

### 5. 데이터 및 일관성 (Data & Consistency)

* [ ] **일관성 모델:** 서비스 간 데이터 정합성은 **즉시 일관성**인가, **최종 일관성(Eventual)**인가?
* [ ] **DB 확장성:** 읽기/쓰기 분리(Replication)나 샤딩(Sharding)이 필요한 규모인가?
* [ ] **데이터 생애 주기:** 데이터 보존 정책(Retention)과 오래된 데이터의 **아카이빙 전략**이 있는가?

### 6. 로그 및 관측성 (Logging & Observability)

* [ ] **로그 저장소:** 접근 기록(Access Log)을 **DB**에 쓸 것인가, 아니면 **분산 로그 저장소(ELK 등)**에 둘 것인가?
* [ ] **분산 트래킹:** 여러 서비스에 걸친 요청 흐름을 추적하기 위한 **Correlation ID**가 설계되었는가?
* [ ] **모니터링:** 시스템 이상 징후를 감지하고 담당자에게 알리는 **경보(Alerting)** 체계가 있는가?

### 7. 보안 및 권한 관리 (Security & Auth)

* [ ] **인증/인가 방식:** JWT(Stateless)를 쓸 것인가, 세션 기반 방식을 쓸 것인가?
* [ ] **권한 세분화:** 권한 관리는 **역할 기반(RBAC)**인가, 아니면 리소스별 **속성 기반(ABAC)**인가?
* [ ] **데이터 암호화:** 개인정보 및 민감 데이터의 **전송 중/저장 시 암호화** 방식은 무엇인가?

### 8. 외부 의존성 및 제약 사항 (External Dependencies)

* [ ] **벤더 록인:** 특정 클라우드나 솔루션에 종속되는 것을 방지할 **추상화 계층**이 있는가?
* [ ] **비용 상한선:** 인프라 가용성을 높이기 위해 투입 가능한 **비용의 한계**는 어디까지인가?
