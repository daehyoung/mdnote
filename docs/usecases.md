# MarkDown Note System Use Cases (Narrative Strategy)

## 1. 개요 (Philosophy)

본 시스템의 유스케이스는 액터와 시스템 간의 상호작용을 **대화식(Dialog)**으로 기술합니다. 각 단계는 액터의 행동에 대한 시스템의 반응을 능동태 현재 시점으로 묘사하며, 기술적 구현의 추적을 위해 **중단점(Interruptible Point / ID)**을 흐름 내부에 명시합니다.

*   중단점은 `{기능명 [추적ID]}` 형식의 제목으로 표기하며, 이는 시스템이 일을 잠시 멈추고 사용자의 다음 행동을 기다리거나 데이터를 안전하게 커밋하는 지점을 의미합니다.

---

## 2. [Goal 1] 지식 탐색 및 소비 (Knowledge Exploration & Consumption)

*   **최종 가치**: 액터는 조직의 지식 자산을 효과적으로 탐색하여 지능적인 업무 의사결정을 지원받는다.
*   **선행 조건**: 시스템은 독자가 사용할 수 있는 유효한 계정과 인증 수단을 갖춘 상태이다.

### 기본 흐름 (Basic Flow)

#### **{로그인 [UC-U-01]}**
1. **액터는** 시스템에 접속하기 위해 자신의 자격 증명을 입력하고 인증을 요청한다.
2. **시스템은** 입력된 정보를 검증하고, 유효한 경우 접근 권한(JWT)을 발급한다.

#### **{문서 목록 조회 [UC-D-01]}**
3. **시스템은** 액터의 권한에 맞는 홈 화면과 카테고리 트리 구조를 구성하여 보여준다.
4. **액터는** 특정 카테고리를 선택하거나 목록에서 특정 문서를 찾는다.
5. **시스템은** 요청된 조건에 맞는 문서 리스트를 페이징하여 제공한다.

#### **{문서 검색 [UC-D-04]}**
6. **액터는** 필요 시 전역 검색 기능을 통해 특정 키워드와 일치하는 문서를 찾는다.
7. **시스템은** 전체 저장소에서 키워드와 매칭되는 결과를 취합하여 보여준다.

#### **{문서 상세 조회 [UC-D-03]}**
8. **액터는** 확인하려는 문서를 선택한다.
9. **시스템은** 해당 문서의 마크다운 전문과 다이어그램을 렌더링하여 화면에 출력한다.

#### **{댓글 등록 [UC-CM-01]}**
10. **액터는** 문서 내용에 대해 질의하거나 의견을 남기기 위해 댓글을 작성한다.
11. **시스템은** 댓글을 저장하고 해당 문서의 대화 스레드에 즉시 반영한다.

### 대안 흐름 (Alternative Flows)
*   2-a. **시스템은** 자격 증명이 올바르지 않으면 인증 실패 메시지를 보내고 다시 로그인을 유도한다.
*   9-a. **시스템은** 액터에게 해당 문서의 읽기 권한이 없음을 확인하면 접근 거부(403)를 통보한다.

## 3. [Goal 2] 지식 생산 및 자산화 (Knowledge Production & Assetization)

*   **최종 가치**: 액터는 개인이 습득한 정보를 정형화된 지식으로 문서화하여 조직 내 공유 가능한 자산으로 전환한다.
*   **선행 조건**: 액터는 시스템에 로그인되어 있으며 문서 작성 권한을 보유한 상태이다.

### 기본 흐름 (Basic Flow)

#### **{문서 생성/초안 [UC-D-02]}**
1. **액터는** 새로운 지식 기록을 시작하기 위해 문서 작성을 시작한다.
2. **시스템은** 새로운 문서 엔티티를 생성하고 고유 번호를 발급하여 편집 모드를 활성화한다.
3. **액터는** 문서의 제목을 정하고 마크다운 에디터에 내용을 작성한다.

#### **{파일 첨부 [UC-D-05]}**
4. **액터는** 본문 내용에 구체적인 근거를 더하기 위해 보조 자료(파일)를 첨부한다.
5. **시스템은** 파일을 안전한 스토리지에 저장하고 본문 내에 삽입 가능한 경로(URL)를 제공한다.

#### **{문서 내용 저장 [UC-D-02-E]}**
6. **액터는** 지식의 분류(카테고리), 키워드(태그), 그리고 공개 범위를 설정한다.
7. **액터는** 작성을 완료하고 지식을 최종 확정(저장)한다.
8. **시스템은** 모든 메타데이터와 본문을 영구 저장소에 반영하고 문서의 현재 상태를 업데이트한다.

### 대안 흐름 (Alternative Flows)
*   8-a. **시스템은** 필수 항목(제목 등)이 누락되었음을 감지하면 저장 작업을 중단하고 액터에게 보완을 요청한다.

## 4. [Goal 3] 시스템 운영 및 보안 거버넌스 (System Governance)

*   **최종 가치**: 관리자는 조직의 지식 생태계가 안전하고 체계적으로 유지될 수 있도록 통제권을 행사한다.
*   **선행 조건**: 시스템은 액터가 관리자(ADMIN) 권한을 가진 것을 확인한 상태이다.

### 기본 흐름 (Basic Flow)

#### **{사용자 목록 관리 [UC-A-01]}**
1. **액터는** 조직 내 사용자의 변동 사항을 반영하여 계정을 관리하기 위해 사용자 목록을 요청한다.
2. **시스템은** 전체 계정의 목록과 현재 상태를 집계하여 제공한다.
3. **액터는** 신규 계정을 생성하거나 기존 사용자의 권한을 변경한다.
4. **시스템은** 변경된 보안 설정을 즉시 적용하고 이력을 남긴다.

#### **{시스템 카테고리 관리 [UC-A-03]}**
5. **액터는** 전사적인 서비스 분류를 위해 공통 카테고리 트리를 수정한다.
6. **시스템은** 새로운 분류 노드를 생성하고 모든 액터에게 동일한 지식 체계를 배포한다.

#### **{시스템 모니터링 [UC-A-05]}**
7. **액터는** 시스템의 리소스 사용량과 지식 생산 지표를 확인한다.
8. **시스템은** 각종 통계 지표를 시각화하여 대시보드에 출력한다.

---

## 5. 후행 조건 (Post-conditions)
*   모든 지식 활동은 시스템 로그에 기록되며, 데이터베이스는 액터의 행위에 따라 최종 정합성 상태를 유지한다.
