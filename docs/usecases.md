# MarkDown Note System Use Cases (Narrative Strategy)

## 1. 개요 (Philosophy)

본 시스템의 유스케이스는 액터와 시스템 간의 상호작용을 **대화식(Dialog)**으로 기술합니다. 각 단계는 액터의 행동에 대한 시스템의 반응을 능동태 현재 시점으로 묘사하며, 기술적 구현의 추적을 위해 **중단점(Interruptible Point / ID)**을 흐름 내부에 명시합니다.

*   중단점은 `{기능명 [추적ID]}` 형식의 제목으로 표기하며, 이는 시스템이 일을 잠시 멈추고 사용자의 다음 행동을 기다리거나 데이터를 안전하게 커밋하는 지점을 의미합니다.

---

## 2. [Goal 1] 지식 탐색 및 소비 (Knowledge Exploration & Consumption)

*   **최종 가치**: 액터는 조직의 지식 자산을 효과적으로 탐색하여 지능적인 업무 의사결정을 지원받는다.
*   **선행 조건**: 시스템은 독자가 사용할 수 있는 유효한 계정과 인증 수단을 갖춘 상태이다.

### 기본 흐름 (Basic Flow)

#### **{로그인 [UC-U-01]}**
*   **필수 정보 (Collaboration Payload)**: 사용자 식별자(ID), 비밀번호(Password)
1. **액터는** 시스템에 접속하기 위해 자신의 자격 증명을 입력하고 인증을 요청한다.
2. **시스템은** 입력된 정보를 검증하고, 유효한 경우 접근 권한(JWT)을 발급한다.

#### **{문서 목록 조회 [UC-D-01]}**
*   **필수 정보 (Collaboration Payload)**: 카테고리 ID (선택), 문서 상태 (선택), 페이징 정보 (Offset, Limit)
3. **시스템은** 액터의 권한에 맞는 홈 화면과 카테고리 트리 구조를 구성하여 보여준다.
4. **액터는** 특정 카테고리를 선택하거나 목록에서 특정 문서를 찾는다.
5. **시스템은** 요청된 조건에 맞는 문서 리스트를 페이징하여 제공한다.

#### **{문서 검색 [UC-D-04]}**
*   **필수 정보 (Collaboration Payload)**: 검색 키워드 (Query String)
6. **액터는** 필요 시 전역 검색 기능을 통해 특정 키워드와 일치하는 문서를 찾는다.
7. **시스템은** 전체 저장소에서 키워드와 매칭되는 결과를 취합하여 보여준다.

#### **{문서 상세 조회 [UC-D-03]}**
*   **필수 정보 (Collaboration Payload)**: 문서 고유 번호 (Document ID)
8. **액터는** 확인하려는 문서를 선택한다.
9. **시스템은** 해당 문서의 마크다운 전문과 다이어그램을 렌더링하여 화면에 출력한다.

#### **{댓글 등록 [UC-CM-01]}**
*   **필수 정보 (Collaboration Payload)**: 문서 ID (Target), 댓글 본문 (Text Content), 작성자 인증 정보
10. **액터는** 문서 내용에 대해 질의하거나 의견을 남기기 위해 댓글을 작성한다.
11. **시스템은** 댓글을 저장하고 해당 문서의 대화 스레드에 즉시 반영한다.

### 대안 흐름 (Alternative Flows)
*   2-a. **시스템은** 자격 증명이 올바르지 않으면 인증 실패 메시지를 보내고 다시 로그인을 유도한다.
*   9-a. **시스템은** 액터에게 해당 문서의 읽기 권한이 없음을 확인하면 접근 거부(403)를 통보한다.

## 3. [Goal 2] 지식 생산 및 자산화 (Knowledge Production & Assetization)

*   **최종 가치**: 액터는 개인이 습득한 정보를 정형화된 지식으로 문서화하여 조직 내 공유 가능한 자산으로 전환한다.
*   **선행 조건 (Context)**: 
    - 시스템에는 지식을 분류할 **카테고리가 최소 하나 이상 등록**되어 있어야 한다.
    - 액터는 시스템에 **로그인**되어 있으며, **MANAGER 이상의 권한**과 **소속 그룹(부서)** 정보가 필수적으로 할당된 상태여야 한다.

### 기본 흐름 (Basic Flow)

#### **{문서 생성/초안 [UC-D-02]}**
*   **필수 정보 (Collaboration Payload)**: 카테고리 ID, 액터 인증 정보 (ID, 그룹, 권한), **미리 업로드된 첨부파일 ID 목록 (선택)**
1. **액터는** 새로운 지식 기록을 위해 문서 작성을 시작하며, 미리 정의된 **카테고리 중 하나를 필수적으로 선택**한다.
2. **시스템은** 선택된 카테고리 정보와 액터의 계정 정보(ID, 그룹, 권한)를 결합하여 **트랜잭션 컨텍스트**를 구성한다.
3. **시스템은** 새로운 문서 엔티티를 생성하고, **함께 전달된 첨부파일 ID가 있다면 이를 문서와 즉시 연결**하며 편집 모드를 활성화한다.
4. **액터는** 문서의 제목을 정하고 마크다운 에디터에 내용을 작성한다.

#### **{파일 첨부 [UC-D-05]}**
*   **필수 정보 (Collaboration Payload)**: 업로드 파일 데이터 (Binary/Blob), 문서 ID (참조/선택)
5. **액터는** 본문 내용에 구체적인 근거를 더하기 위해 보조 자료(파일)를 첨부한다.
6. **시스템은** 파일을 안전한 스토리지에 저장하고, 해당 파일을 본문에 삽입할 수 있는 **고유 경로(URL)**를 즉시 제공한다. (이 시점에 파일은 독립적으로 존재하며, 최종 저장 시 문서와 연결된다.)

#### **{문서 내용 저장 [UC-D-02-E]}**
*   **필수 정보 (Collaboration Payload)**: 문서 제목 (Title), 마크다운 본문 (Markdown), 태그 목록 (Tags), 첨부파일 ID 목록 (Attachment IDs), 보안 설정
7. **액터는** 키워드(태그)를 입력하고, 필요 시 기본 설정된 공개 범위를 조정한다.
8. **액터는** 작성을 완료하고 지식을 최종 확정(저장)한다.
9. **시스템은** 액터 정보, 카테고리, 본문, 태그, **업로드된 첨부파일들** 및 타임스탬프를 포함한 전체 **트랜잭션 데이터 세트**를 영구 저장소에 원자적(Atomic)으로 반영한다. (이 단계에서 문서 ID가 최종 생성되거나 확정되며 모든 구성요소가 연결된다.)

### 대안 흐름 (Alternative Flows)
*   8-a. **시스템은** 필수 항목(제목 등)이 누락되었음을 감지하면 저장 작업을 중단하고 액터에게 보완을 요청한다.

## 4. [Goal 3] 시스템 운영 및 보안 거버넌스 (System Governance)

*   **최종 가치**: 관리자는 조직의 지식 생태계가 안전하고 체계적으로 유지될 수 있도록 통제권을 행사한다.
*   **선행 조건**: 시스템은 액터가 관리자(ADMIN) 권한을 가진 것을 확인한 상태이다.

### 기본 흐름 (Basic Flow)

#### **{사용자 목록 조회 [UC-A-01A]}**
*   **필수 정보 (Collaboration Payload)**: 필터 조건 (부서 ID, 역할 등), 페이징 파라미터(Offset, Limit)
1. **액터는** 조직 내 사용자의 변동 사항을 파악하고 관리하기 위해 사용자 목록을 요청한다.
2. **시스템은** 전체 계정의 목록과 현재 상태를 집계하여 제공한다.

#### **{사용자 등록/수정 [UC-A-01B]}**
*   **필수 정보 (Collaboration Payload)**: 사용자 ID (Login ID), 비밀번호, 이메일, 성명, 역할(ADMIN/USER/MANAGER), 부서 정보 (Dept ID)
3. **액터는** 신규 계정을 생성하거나 기존 사용자의 정보 및 권한을 변경한다.
4. **시스템은** 입력된 정보를 검증하고 변경된 보안 설정을 즉시 반영하며 관리 이력을 남킨다.

#### **{시스템 카테고리 관리 [UC-A-03]}**
*   **필수 정보 (Collaboration Payload)**: 카테고리 이름 (Category Name), 부모 카테고리 식별자 (Parent Category ID / Nullable)
5. **액터는** 전사적인 서비스 분류를 위해 공통 카테고리 트리를 수정한다.
6. **시스템은** 새로운 분류 노드를 생성하고 모든 액터에게 동일한 지식 체계를 배포한다.

#### **{시스템 모니터링 [UC-A-05]}**
*   **필수 정보 (Collaboration Payload)**: 조회 기간 (Start Date, End Date), 모니터링 대상 지표 (Resource/Knowledge Metrics)
7. **액터는** 시스템의 리소스 사용량과 지식 생산 지표를 확인한다.
8. **시스템은** 각종 통계 지표를 시각화하여 대시보드에 출력한다.

#### **{시스템 자원 최적화 [UC-A-06]}**
*   **필수 정보 (Collaboration Payload)**: 삭제 대상 파일 ID 목록 (Orphaned File IDs)
9. **액터는** 시스템의 저장 공간 효율화를 위해 연결된 문서가 없는 **고립된 첨부파일(Orphaned Files) 목록**을 요청한다.
10. **시스템은** 데이터베이스를 스캔하여 `doc_id`가 비어있고 생성된 지 일정 시간이 지난 파일 목록을 식별하여 제공한다.
11. **액터는** 불필요하다고 판단되는 파일을 일괄 삭제 요청한다.
12. **시스템은** 대상 파일의 데이터베이스 레코드와 물리 저장소의 데이터를 완전히 제거한다.

---

## 5. 후행 조건 (Post-conditions)
*   모든 지식 활동은 시스템 로그에 기록되며, 데이터베이스는 액터의 행위에 따라 최종 정합성 상태를 유지한다.
